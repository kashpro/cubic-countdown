Browser support

Chrome, Firefox, Safari, Opera, Edge: full support

IE 11-: not supported

Usage

1. Copy CubicCountdown.js and CubicCountdown.css files from "CubicCountdown" folder to your project.

2. Add links in the <head> section in your HTML file:

<html>
    <head>
        ...
        <link rel="stylesheet" href="css/CubicCountdown.css">
        <script src="js/CubicCountdown.js"></script>
        ...
    </head>
    <body>
        ...
    </body>
</html>

3. Place an empty <div> somwhere in the <body> section:

<html>
    <head>
        ...
        <link rel="stylesheet" href="css/CubicCountdown.css">
        <script src="js/CubicCountdown.js"></script>
        ...
    </head>
    <body>
        ...
        <div id="cubic-container"></div>
        ...
    </body>
</html>

4. Initialize your countdown in your script file or an inline <script> tag:

<html>
    <head>
        ...
        <link rel="stylesheet" href="css/CubicCountdown.css">
        <script src="js/CubicCountdown.js"></script>
        ...
    </head>
    <body>
        ...
        <div id="cubic-container"></div>
        ...
        <script>
          var myCount = new Cubic({
              element: "#cubic-container",
              ...more options...
          });
        </script>
    </body>
</html>

All options, passed to Cubic() function must be wrapped with { }.

Only "element" option is required. Minimum initialization:

var myCount = new Cubic({
    element: "#cubic-container",
});

Countdown initialization with full list of options (example):

var myCount = new Cubic({
    element: "#cubic-container",
    toTime: "2025-01-01T00:00:00+08:00",
    cssClass: "zzcubic",
    cubeSize: 160,
    cubeSideMargin: 20,
    cubeTextSize: 100,
    colonSize: 17,
    labelTextSize: 20,
    labelOnTop: false,
    labelOffset: 40,
    daysLabel: "days",
    hoursLabel: "hours",
    minutesLabel: "minutes",
    secondsLabel: "seconds",
    showDays: true,
    showHours: true,
    showMinutes: true,
    showSeconds: true,
    leadingZero: true,
    shadowColor: "#414141",
    shadowIntensity: 100,
    animationPreset: 0,
    animationDelay: 100,
    continuousAnimation: false,
    colonAnimation: true,
    autoStart: true,
    onFinish: function(){
        alert("Happy New Year!");
    },
    onTick: function(){
        console.log("tick");
    },
    mobileFirst: false,
    responsive: [
        {
            breakpoint: 1200,
            options: {
                ...options...
            },
        },
        {
            breakpoint: 768,
            options: {
                ...options...
            },
        },
    ],
});

element

element: "#cubic-container",

Defines a DOM-element, where the countdown will be mounted. Value must be a valid CSS-selector (#selector - for id, .selector - for class). Required parameter.

If this parameter is not specified or the value is not a valid CSS-selector, a warning (Cubic Countdown: Incorrect selector) will be displayed in the browser console.

If an element with the specified selector is missing on the page, a warning (Cubic Countdown: Element not found) will be displayed in the browser console.

toTime

Displayed time is equal to the difference between user's local time and destination time specified in the "toTime" parameter. If user's local time lesser than destination time, the countdown will count down (to time in future). If user's local time more than destination time, the countdown will count up (from time in past).

The countdown will continue counting when destination time is reached. You can stop timer via onFinish callback.

There are two ways to specify this option: string and Date object.

1. String (use to specify a certain time).

toTime: "2025-01-01T00:00:00+08:00",

"T" symbol between date and time sections is required (for Edge browser compatibility).

You can specify desired timezone in the end of the string. For example:

toTime: "2025-01-01T00:00:00+08:00", //(Australia)

Timezone range: -23:59 ... +23.59.

To specify UTC timezone, you can add "Z" symbol to the end of the string:

toTime: "2025-01-01T00:00:00Z",

or add "+00:00":

toTime: "2025-01-01T00:00:00+00:00",

If the timezone is not specified user's local timezone will be taken:

toTime: "2025-01-01T00:00:00",

Usage examples:

New Year countdown (Local date for each user). Specify "toTime" option in user's local time (without timezone):

toTime: "2025-01-01T00:00:00",

Site maintenance countdown (Global date for each user). Specify "toTime" option with your (or your site) timezone:

toTime: "2025-01-01T00:00:00+08:00",

Count up from New Year (Local date for each user). Specify "toTime" option with date in past, without timezone:

toTime: "2010-01-01T00:00:00",

Count up from foundation of the company (Global date for each user). Specify "toTime" option with date in past, with your timezone:

toTime: "2010-01-01T00:00:00+08:00",

Warning! To display large values of the number of days you may have to reduce the font size.

Use as a clock: show user's current local time. Turn off "days" cube visibility, specify "toTime" option with any date in past, specify time with "00:00:00" without timezone:

toTime: "2000-01-01T00:00:00",

See clock.html in "Examples" folder.

2. Date object (use to specify time relative to the user's current local time).

Usage examples:

60 seconds (1 minute) countdown since page loaded. The countdown will count from 00:00:01:00 to zero:

toTime: Date.now() + 60 * 1000,

60 minutes (1 hour) countdown since page loaded. The countdown will count from 00:01:00:00 to zero:

toTime: Date.now() + 60 * 60 * 1000,

24 hours (1 day) countdown since page loaded. The countdown will count from 01:00:00:00 to zero:

toTime: Date.now() + 24 * 60 * 60 * 1000,

Count up since page loaded. The countdown will count from 00:00:00:00 to infinity:

toTime: Date.now(),

Count up with offset since page loaded. The countdown will count from 00:00:01:00 to infinity:

toTime: Date.now() - 60 * 1000,

Optional parameter. Default value: "Date.now() + 1 day".

This parameter can also be generated dynamically on the server side.

cssClass

cssClass: "zzcubic",

Defines a CSS class prefix for all DOM-elements of the countdown. Useful for styling.

Optional parameter. Default value: "zzcubic".

cubeSize

cubeSize: 150,

Defines width and height of the cude (in pixels). This parameter always transforms to even value (75 -> 74).

Optional parameter. Default value: 150.

cubeSideMargin

cubeSideMargin: 20,

Defines left and right margins of the cude (in pixels).

Optional parameter. Default value: 20.

cubeTextSize

cubeTextSize: 100,

Defines font size of the cube faces.

Optional parameter. Default value: 100.

colonSize

colonSize: 20,

Defines width and height of the colon dots.

If you don't need colons, set colonSize: 0.

Optional parameter. Default value: 20.

labelTextSize

labelTextSize: 20,

Defines font size of the labels.

If you don't need labels, set labelTextSize: 0.

Optional parameter. Default value: 20.

labelOnTop

labelOnTop: true/false,

Defines label position: true - above the cubes, false - under the cubes.

Optional parameter. Default value: false.

labelOffset

labelOffset: 40,

Defines labels vertical offset.

If you don't need labels, set labelOffset: 0.

Optional parameter. Default value: 40.

daysLabel, hoursLabel, minutesLabel, secondsLabel

daysLabel: "days",
hoursLabel: "hours",
minutesLabel: "minutes",
secondsLabel: "seconds",

Defines labels text.

Optional parameters. Default values: "days", "hours", "minutes", "seconds".

showDays, showHours, showMinutes, showSeconds

showDays: true/false,
showHours: true/false,
showMinutes: true/false,
showSeconds: true/false,

Defines visbility of the cubes and labels. Colons visibility defines automatically.

Optional parameters. Default values: true for all cubes.

leadingZero

leadingZero: true/false,

Defines leading zero if cube value < 10.

Optional parameter. Default value: true.

shadowColor

shadowColor: "#414141",

Defines shadow color while cube rotating.

Optional parameter. Default value: "#414141".

shadowIntensity

shadowIntensity: 100,

Defines shadow intensity while cube rotating.

If you don't need shadows, set shadowIntensity: 0.

Optional parameter. Default value: 100. Max value: 100.

animationPreset

animationPreset: 0/1/2/3,

Defines the direction of cube rotate:

  • 0: top-to-bottom
  • 1: bottom-to-top
  • 2: left-to-right
  • 3: right-to-left

Optional parameter. Default value: 0. Max value: 3.

animationDelay

animationDelay: 50,

Defines the delay of the rotation animation (in milliseconds) between adjacent cubes.

Optional parameter. Default value: 50. Max value: 100.

continuousAnimation

continuousAnimation: true/false,

If true, all cubes will rotate every second while counting, even if the value of the cube does not change.

Optional parameter. Default value: false.

colonAnimation

colonAnimation: true/false,

If false, all colons will not be animated.

Optional parameter. Default value: true.

autoStart

autoStart: true/false,

If true, the countdown will start counting immediately after initialization.

Optional parameter. Default value: true.

onFinish

onFinish: function(){
    ...your code here...
},

Defines user callback function that will be executed when the countdown time will become "00:00:00:00".

Example:

onFinish: function(){
    myCount.stop(); //see methods below
    alert("Happy New Year!");
},

Optional parameter. Default value: empty function.

See callbacks.html in "Examples" folder.

onTick

onTick: function(){
    ...your code here...
},

Defines user callback function that will be executed when the countdown ticks.

Example:

onTick: function(){
    console.log("tick");
},

Optional parameter. Default value: empty function.

See callbacks.html in "Examples" folder.

mobileFirst

mobileFirst: true/false,

Defines, how to interpret "responsive" parameter (see below).

If false all responsive rules will interpret like "@media (max-width: value)."

If true all responsive rules will interpret like "@media (min-width: value)."

Optional parameter. Default value: false.

responsive

responsive: [
    {
        breakpoint: value in pixels,
        options: {
            ...options...
        },
    },
    {
        breakpoint: value in pixels,
        options: {
            ...options...
        },
    },
    ...more rules...
],

Array with adaptive rules. These rules works like CSS media queries.

Usage examples:

1. mobileFirst = false:

mobileFirst: false,
/* if viewport width > 1200:
cubeSize = 150, cubeTextSize = 100 */
cubeSize: 150,
cubeTextSize: 100,
responsive: [
    /* if viewport width 769 ... 1200:
    cubeSize = 120, cubeTextSize = 80 */
    {
        breakpoint: 1200,
        options: {
            cubeSize: 120,
            cubeTextSize: 80,
        },
    },
    /* if viewport width 541 ... 768:
    cubeSize = 75, cubeTextSize = 50 */
    {
        breakpoint: 768,
        options: {
            cubeSize: 75,
            cubeTextSize: 50,
        },
    },
    /* if viewport width <= 540:
    cubeSize = 50, cubeTextSize = 30 */
    {
        breakpoint: 540,
        options: {
            cubeSize: 50,
            cubeTextSize: 30,
        },
    },
],

2. mobileFirst = true:

mobileFirst: true,
/* if viewport width < 540:
cubeSize = 150, cubeTextSize = 100 */
cubeSize: 150,
cubeTextSize: 100,
responsive: [
    /* if viewport width 540 ... 767:
    cubeSize = 50, cubeTextSize = 30 */
    {
        breakpoint: 540,
        options: {
            cubeSize: 50,
            cubeTextSize: 30,
        },
    },
    /* if viewport width 768 ... 1199:
    cubeSize = 75, cubeTextSize = 50 */
    {
        breakpoint: 768,
        options: {
            cubeSize: 75,
            cubeTextSize: 50,
        },
    },
    /* if viewport width >= 1200:
    cubeSize = 120, cubeTextSize = 80 */
    {
        breakpoint: 1200,
        options: {
            cubeSize: 120,
            cubeTextSize: 80,
        },
    },
],

Optional parameter. Default value: none.

See responsive.html in "Examples" folder.

Methods

After the countdown initialization some methods are available:

start

myCount.start();

Start count. Works only if the countdown is stopped.

stop

myCount.stop();

Stop count. Works only if the countdown is running.

remove

myCount.remove();

Remove the countdown from DOM.

restore

myCount.restore();

Append the countdown to DOM.

hide

myCount.hide();

Hide the countdown on the page (set "display: none" rule).

show

myCount.show();

Show the countdown on the page (remove "display: none" rule).

onFinish

myCount.onFinish();

Execute "onFinish" callback, specified in onFinish parameter.

onTick

myCount.onTick();

Execute "onTick" callback, specified in onTick parameter.

getTime

var time = myCount.getTime();
/*
{
    dd: "23",
    hh: "59",
    mm: "59",
    ss: "30"
}
*/

Get object with current time values.

change

myCount.change({
    ...options...
});

Change options on the fly.

CSS styling

Visual properties that are not affected by javaScript code (such as face color, font family, font color, border, etc.) are contained in a CSS file.

See CubicCountdown/css/CubicCountdown.css file. It is well commented, feel free to change, add or remove any CSS styles and rules.

If you need more than one countdown with different visual styles on one page, initialize countdowns with a different cssClass parameter.

See two-countdowns.html and two-countdowns.css in "Examples" folder.
